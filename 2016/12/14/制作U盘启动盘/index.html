制作U盘启动盘

发表于 2016-12-14   |   分类于 LinuxSystem   |  

由Kickstart制作的ISO镜像直接由UltraISO工具写入U盘制作启动盘，安装系统的时候会报错mount: no medium found on /dev/sr0。通过搜索得知由Kickstart工具制作的ISO不能直接写入U盘进行安装，需要执行一系列操作，下面是对操作的总结。

准备工作

  1. 一个由Kickstart工具制作的ISO镜像文件
  2. 一个U盘，格式为FAT32，至少8G大小
  3. 一个正常运行的Linux操作系统

制作U盘安装盘

我所使用的U盘在系统中显示为/dev/sdb，下面将以/dev/sdb为示例来说明整个过程：

首先格式化U盘

mkfs命令可以用来格式化磁盘，针对不同磁盘类型需要指定相对应的类型参数，如mkfs -t vfat /dev/sdb。同时mkfs针对不同类型的磁盘格式有对应的格式化命令，如：mkfs.vfat mkfs.ext2 mkfs.ext3 mkfs.xfs等

下面使用mkfs.vfat格式化U盘：

mkfs.vfat /dev/sdb

执行以上命令可能会报错：mkfs.vfat:Device partition expected, not making filesystem on entire device ‘/dev/sdb’ (use -I to override)
如果出现此错误，则执行如下命令：

mkfs.vfat -I /dev/sdb

为U盘创建分区

为U盘创建两个分区：

  • 一个类型为W95 FAT32(LBA)(可以通过fdisk命令的c参数指定)，大小为500M，此分区作为启动分区
  • 将剩余空间创建为一个ext3类型的分区

下面使用fdisk命令对磁盘进行分区：

使用mkfs格式化分区:

$ sudo mkfs -t vfat -n "BOOT" /dev/sdb1
$ sudo mkfs -L "DATA" /dev/sdb2

将MBR数据写入设备

$ sudo dd conv=notrunc bs=440 count=1 \
           if=/usr/share/syslinux/mbr.bin of=/dev/sdb

将syslinux安装到第一个分区

$ sudo syslinux /dev/sdb1

拷贝文件到U盘

  1. 挂载分区和ISO镜像：

    mkdir BOOT && sudo mount /dev/sdb1 BOOT
mkdir DATA && sudo mount /dev/sdb2 DATA
mkdir DVD && sudo mount -o loop /root/kickstart.iso DVD

  2. 拷贝DVD/isolinux/目录下的内容到BOOT/目录下：

    sudo cp DVD/isolinux/* BOOT

  3. 重命名isolinux.cfg为syslinux.cfg:

    sudo mv BOOT/isolinux.cfg BOOT/syslinux.cfg

  4. 拷贝kickstart.iso到DATA目录下：

    sudo cp /root/kickstart.iso DATA/

  5. 修改syslinux.cfg文件：
    在syslinux.cfg文件修改前的内容为：

    label linux
  menu label ^Install CentOS7
  kernel vmlinuz
  append initrd=initrd.img inst.stage2=hd:LABEL=CentOS\x207\x20x86_64 \
        inst.ks=cdrom:/dev/cdrom:/isolinux/ks.cfg

    修改后syslinux.cfg相对应的内容为：

    label linux
  menu label ^Install CentOS7
  kernel vmlinuz
  append initrd=initrd.img inst.stage2=hd:LABEL=DATA:/ inst.ks=hd:LABEL=BOOT:/ks.cfg

  1. 如果需要支持EFI启动，则同时还需要将DVD/{images,EFI}目录下的内容拷贝到BOOT目录下

    sudo cp DVD/{images,EFI} BOOT/ -r

  2. 修改BOOT/EFI/BOOT/grub.cfg文件：
    文件grub.cfg修改前，相关内容为：

    search --no-floppy --set=root -l 'CentOS 7 x86_64'

### BEGIN /etc/grub.d/10_linux ###
menuentry 'Install RiverCloudSystem' --class fedora \
    --class gnu-linux --class gnu --class os {
        linuxefi /images/pxeboot/vmlinuz \
    inst.stage2=hd:LABEL=CentOS\x207\x20x86_64 \
    inst.ks=cdrom:/dev/cdrom:/isolinux/ks.cfg
        initrdefi /images/pxeboot/initrd.img
}

    文件grub.cfg修改后，相关内容为：

    search --no-floppy --set=root -l 'BOOT'                                   

### BEGIN /etc/grub.d/10_linux ###                                        
menuentry 'Install RiverCloudSystem' --class fedora \
    --class gnu-linux --class gnu --class os {
        linuxefi /images/pxeboot/vmlinuz \
    inst.stage2=hd:LABEL=DATA:/ \
    inst.ks=hd:LABEL=BOOT:/ks.cfg 
        initrdefi /images/pxeboot/initrd.img
}
  3. 在BOOT/EFI/BOOT目录下创建一个名为startup.nsh的文件，其内只有一行，内容为：BOOTX64.

注意: 在支持EFI启动的ISO中，必须包括以下几个软件包：grub2-efi-2.02-0.29.el7.centos.x86_64.rpm,efibootmgr-0.8.0-7.el7.x86_64.rpm,efivar-libs-0.11-1.el7.x86_64.rpm,shim-0.9-2.el7.x86_64.rpm，否则在进行ISO安装的时候会报错
